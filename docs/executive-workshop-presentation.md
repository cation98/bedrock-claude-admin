### 슬라이드 2: 왜 Claude Code인가?

**핵심 메시지**: AI 코딩 어시스턴트의 시대, 보안과 생산성을 동시에

```
┌─────────────────────────────────────────────────────┐
│                   AI 코딩 도구 비교                    │
├──────────────┬──────────┬──────────┬────────────────┤
│              │ ChatGPT  │ GitHub   │ Claude Code    │
│              │ Web      │ Copilot  │ (사내 구축)     │
├──────────────┼──────────┼──────────┼────────────────┤
│ 데이터 보안   │ ✗ 외부전송 │ ✗ 외부전송 │ ✓ 사내 폐쇄망   │
│ 사내 DB 접근  │ ✗ 불가   │ ✗ 불가    │ ✓ RDS 직접 조회 │
│ 파일 업로드   │ △ 제한적  │ ✗ 없음    │ ✓ 드래그&드롭   │
│ 코드 실행     │ △ 샌드박스 │ ✗ 제안만  │ ✓ 실제 환경     │
│ 사내 SSO      │ ✗ 별도계정 │ ✗ 별도계정 │ ✓ SSO 통합     │
│ 비용 통제     │ ✗ 개인과금 │ ✗ 라이선스 │ ✓ AWS Bedrock  │
└──────────────┴──────────┴──────────┴────────────────┘
```

**포인트**: "사내 데이터가 한 바이트도 외부로 나가지 않습니다"

### 슬라이드 3: 보안 아키텍처 — 데이터가 외부로 나가지 않는 구조

**차트: 네트워크 보안 다이어그램**

```
                    ┌─────────────────────────┐
                    │      인터넷 차단 구간      │
                    │   (VPC Private Subnet)   │
                    │                          │
  사용자 브라우저 ──→│  ┌──────────────────┐    │
  (사내망 Only)     │  │  EKS 클러스터     │    │
                    │  │  ┌──────────┐    │    │
  SSO 인증 ────────→│  │  │사용자 Pod │    │    │──→ AWS Bedrock
  (sso.skons.net)   │  │  │(격리 환경) │    │    │   (Claude API)
                    │  │  └──────────┘    │    │   ┌─────────────┐
                    │  └──────────────────┘    │   │ VPC Endpoint │
                    │         │                │   │ (사내→Bedrock)│
                    │         ▼                │   └─────────────┘
                    │  ┌──────────────────┐    │
                    │  │  사내 RDS        │    │  ※ 외부 인터넷 통신 없음
                    │  │  (고장 데이터 등)  │    │  ※ API Key 노출 위험 없음
                    │  └──────────────────┘    │  ※ 사용자간 격리 보장
                    └─────────────────────────┘
```

**보안 포인트 (불렛)**:
- AWS VPC Private Subnet 내 운영 — 외부 인터넷 직접 통신 없음
- AWS Bedrock VPC Endpoint — AI 모델 호출도 사내망 내부 경로
- 사용자별 Pod 격리 — 다른 사용자의 데이터 접근 불가 (NetworkPolicy)
- SSO 통합 인증 — 별도 회원가입 불필요, 기존 사번으로 로그인
- ReadOnly DB 접근 — 조회만 가능, 데이터 변경/삭제 불가

### 슬라이드 8: 실습 시나리오 1 — 네트워크 고장 현황 분석

**시나리오 제목**: "현재 네트워크 고장 현황을 AI로 분석하기"

```
실습 단계:

  Step 1                Step 2                Step 3
  ┌──────────┐          ┌──────────┐          ┌──────────┐
  │ 로그인    │   ──→    │ claude   │   ──→    │ AI에게    │
  │ SSO 인증  │          │ 명령 입력 │          │ 질문하기  │
  └──────────┘          └──────────┘          └──────────┘

  Step 4                Step 5
  ┌──────────┐          ┌──────────┐
  │ 결과 확인 │   ──→    │ 보고서   │
  │ 차트/표   │          │ 다운로드 │
  └──────────┘          └──────────┘
```

**실습 지시문 예시**:
```
"psql-tango로 접속해서 현재 활성 고장 현황을 팀별로 조회하고,
 최근 24시간 고장 추이를 시간대별로 분석해서 차트로 만들어줘.
 결과를 엑셀 파일로 저장해줘."
```

**기대 결과**:
- 팀별 고장 건수 표
- 시간대별 추이 차트 (matplotlib)
- 엑셀 파일 자동 생성 → 다운로드

### 슬라이드 10: 실습 시나리오 3 — 보고서 자동 생성

**시나리오 제목**: "DB 데이터로 보고서 자동 작성"

**실습 지시문 예시**:
```
"안전관리 DB에서 최근 1개월 점검 현황을 조회하고,
 부서별 점검 완료율을 계산해서
 임원 보고용 요약 보고서를 작성해줘.
 결과를 엑셀로 저장해줘."
```

**AI가 수행하는 작업**:
```
  ┌─────────────┐
  │ 1. DB 쿼리   │  psql-safety로 데이터 조회
  ├─────────────┤
  │ 2. 데이터    │  pandas로 집계/분석
  │    분석      │
  ├─────────────┤
  │ 3. 차트 생성 │  matplotlib 시각화
  ├─────────────┤
  │ 4. 보고서    │  요약문 자동 작성
  │    작성      │
  ├─────────────┤
  │ 5. 파일 저장 │  엑셀 + 차트 이미지
  └─────────────┘

  전체 소요 시간: 약 1~2분
  (수동 작업 시: 30분~1시간)
```

## 실습 진행 시나리오 (진행자용)

### 사전 준비 (실습 30분 전)

```bash
# 1. Admin Dashboard에서 참석자 세션 일괄 생성
#    워크숍 관리 → 사번 목록 입력 → "전체 세션 생성"

# 2. Pod 생성 확인
kubectl get pods -n claude-sessions
# 모든 Pod이 Running 상태인지 확인

# 3. TANGO DB 데이터 확인
psql-tango -c "SELECT COUNT(*) FROM alarm_data;"
# 현재 활성 알람 건수 확인
```

### 실습 진행 순서 (60분)

```
시간     내용
─────    ──────────────────────────────────
0~10분   플랫폼 소개 + 보안 설명 (슬라이드 1~7)
10~15분  로그인 + 환경 익히기 (슬라이드 12)
15~25분  시나리오 1: 고장 현황 분석 (슬라이드 8)
25~35분  시나리오 2: 엑셀 파일 분석 (슬라이드 9)
35~45분  시나리오 3: 보고서 자동 생성 (슬라이드 10)
45~55분  자유 질문 + 실습 (슬라이드 11)
55~60분  Q&A + 마무리 (슬라이드 15)
```

### 실습 중 자주 나오는 질문

| 질문 | 답변 |
|------|------|
| "데이터가 외부로 나가나요?" | 아닙니다. VPC 내부 + VPC Endpoint로 100% 사내 경유 |
| "비용은 얼마나 드나요?" | AWS Bedrock 종량제, 1회 질문당 약 10~50원 수준 |
| "다른 사람 데이터를 볼 수 있나요?" | 불가합니다. Pod별 NetworkPolicy로 격리 |
| "세션이 끝나면 데이터는?" | Pod 삭제 시 모든 데이터 자동 소멸 |
| "어떤 질문이든 가능한가요?" | 코딩, 분석, 번역, 요약 등 텍스트 기반 업무 모두 가능 |
