## 1. 실습 개요

### Claude Code란?

Claude Code는 **터미널(명령줄)에서 동작하는 AI 코딩 어시스턴트**입니다. 채팅창에 자연어로 요청하면 AI가 다음과 같은 작업을 직접 수행합니다.

| 할 수 있는 일 | 예시 |
|--------------|------|
| 코드 분석 및 설명 | "이 프로젝트의 구조를 설명해줘" |
| 데이터베이스 조회 및 분석 | "최근 한 달간 안전 점검 통계를 뽑아줘" |
| 새 코드 작성 | "월별 추이를 보여주는 API를 만들어줘" |
| 문서 작성 | "이 코드의 API 문서를 한국어로 만들어줘" |
| 업무 자동화 설계 | "주간 보고서 자동 생성 시스템을 설계해줘" |

### 오늘의 목표

> **AI가 실제 업무 환경에서 어떻게 활용될 수 있는지 직접 체험합니다.**

- 코드를 몰라도 괜찮습니다. 한국어로 질문하면 됩니다.
- 정해진 정답이 없습니다. 자유롭게 질문해 보세요.
- 모든 데이터는 사내 AWS 환경 안에서만 처리됩니다.

### 실습 환경 구성

오늘 여러분이 사용하는 환경에는 다음 도구가 미리 설치되어 있습니다.

| 도구 | 용도 |
|------|------|
| Claude Code CLI | AI 코딩 어시스턴트 (오늘의 주인공) |
| psql | 데이터베이스 조회 도구 |
| Python 3 | 데이터 분석 및 스크립트 실행 |
| git | 코드 버전 관리 |
| 샘플 프로젝트 | FastAPI 기반 안전관리 시스템 (~/workspace/sample-project/) |

## 2. 접속 방법

### 단계 1: 웹 브라우저 열기

아무 웹 브라우저(Chrome, Edge, Safari)를 엽니다.

### 단계 2: 접속 URL 입력

관리자가 안내하는 URL을 주소창에 입력합니다.

```
https://[관리자가 안내하는 URL]
```

### 단계 3: 로그인

사내 SSO 계정(사번)으로 로그인합니다.

### 단계 4: 터미널 화면 확인

로그인에 성공하면 아래와 같은 검은색 터미널 화면이 나타납니다.

```
====================================
  Claude Code 실습 환경에 오신 것을 환영합니다!
====================================

사용 가능한 도구: claude, psql, python3, git
샘플 프로젝트: ~/workspace/sample-project/

시작하려면 'claude' 를 입력하세요.

user@claude-terminal:~$
```

### 단계 5: Claude Code 동작 확인

터미널에 아래 명령어를 입력하여 Claude Code가 정상적으로 설치되어 있는지 확인합니다.

```bash
claude --version
```

아래와 비슷한 결과가 나오면 정상입니다.

```
1.x.x (claude-code)
```

> **💡 팁:** 터미널에 텍스트를 입력할 때는 마우스로 터미널 화면을 한 번 클릭한 뒤 키보드로 타이핑합니다. 입력 후 Enter 키를 누르면 명령이 실행됩니다.

### 시나리오 1: 코드 이해하기 (15분)

**목표**: Claude에게 프로젝트 코드를 분석하고 설명하도록 요청합니다.

> 코드를 전혀 모르더라도 Claude가 쉽게 설명해 줍니다. 마치 개발자에게 "이 코드가 뭐 하는 거야?"라고 묻는 것과 같습니다.

#### 순서

**1단계: Claude Code 시작**

터미널에 다음을 입력합니다.

```bash
claude
```

몇 초 후 Claude Code 대화 화면이 나타납니다.

```
╭──────────────────────────────────────╮
│ Claude Code                          │
│                                      │
│ /help for available commands         │
╰──────────────────────────────────────╯

>
```

> **💡 팁:** `>` 표시가 나타나면 Claude에게 질문할 준비가 된 것입니다.

**2단계: 프로젝트 구조 분석 요청**

아래 문장을 `>` 뒤에 그대로 입력하고 Enter를 누릅니다.

```
~/workspace/sample-project/ 프로젝트의 구조를 분석하고 설명해줘
```

**예상 결과**: Claude가 프로젝트의 파일과 폴더를 읽은 뒤, 아래와 같은 내용을 설명합니다.

- 프로젝트가 FastAPI 프레임워크를 사용한다는 점
- `app/main.py`가 애플리케이션의 진입점이라는 점
- `app/routes/safety.py`에 안전 점검 관련 API가 정의되어 있다는 점
- `app/models.py`에 데이터베이스 모델이 있다는 점

> Claude가 파일을 읽을 때 "도구를 사용해도 되겠습니까?" 같은 확인 메시지가 나올 수 있습니다. **'y'를 입력하고 Enter**를 누르면 됩니다.

**3단계: 특정 코드 리뷰 요청**

이어서 아래를 입력합니다.

```
app/routes/safety.py 코드를 리뷰해줘. 어떤 API가 있고, 개선할 점은 없는지 알려줘.
```

**예상 결과**: Claude가 해당 파일을 읽고 다음을 설명합니다.

- 3개의 API 엔드포인트 (목록 조회, 상세 조회, 통계)가 있다는 점
- 각 API가 하는 일을 비개발자도 이해할 수 있게 설명
- 에러 처리, 페이징, 보안 등 개선할 수 있는 점을 제안

**4단계: 비개발자용 설명 요청 (선택)**

```
이 프로젝트가 하는 일을 비개발자도 이해할 수 있게 한 문단으로 요약해줘
```

> **💡 팁:** Claude에게 "비개발자도 이해할 수 있게"라고 요청하면 기술 용어 없이 쉽게 설명해 줍니다.

### 시나리오 2: DB 데이터 분석 (20분)

**목표**: Claude에게 실제 데이터베이스에 접속하여 데이터를 조회하고 분석하도록 요청합니다.

> 이 환경은 실제 safety-prod 데이터베이스의 **읽기 전용(ReadOnly)** 복제본에 연결되어 있습니다. 데이터를 조회만 할 수 있고 수정할 수 없으므로 안전합니다.

#### 순서

**1단계: 데이터베이스 테이블 확인**

Claude에게 아래와 같이 요청합니다. (이전 시나리오에서 Claude Code가 이미 실행 중이라면 바로 입력합니다.)

```
safety-prod DB에 접속해서 어떤 테이블이 있는지 확인해줘. psql $DATABASE_URL 명령어로 접속할 수 있어.
```

**예상 결과**: Claude가 `psql` 명령어를 실행하여 데이터베이스에 접속하고, 테이블 목록을 보여줍니다.

- 테이블 이름과 각 테이블이 어떤 데이터를 담고 있는지 설명
- 주요 컬럼(열)의 의미를 알기 쉽게 정리

**2단계: 데이터 통계 분석**

```
최근 한 달간 안전 점검 보고서 통계를 분석해줘. 총 건수, 심각도별 분포, 처리 상태별 현황을 알려줘.
```

**예상 결과**: Claude가 SQL 쿼리를 작성하고 실행하여 다음과 같은 분석 결과를 보여줍니다.

- 최근 한 달간 총 보고서 수
- 심각도(severity)별 건수: low, medium, high, critical
- 상태(status)별 건수: open, in_progress, resolved
- 처리율(resolution rate) 계산

**3단계: 시각화 코드 생성**

```
심각도별 보고서 현황을 막대 차트로 표현하는 Python 코드를 만들어줘
```

**예상 결과**: Claude가 matplotlib 등을 사용한 Python 코드를 생성하고, 원하면 직접 실행하여 차트 이미지 파일을 생성합니다.

**4단계: 심화 분석 (선택)**

아래 중 관심 있는 것을 요청해 보세요.

```
지역(location)별로 보고서 건수를 비교해줘
```

```
최근 6개월간 월별 보고서 추이를 분석해줘
```

```
미해결(open) 보고서 중 critical인 것들을 목록으로 보여줘
```

> **⚠️ 주의:** 데이터베이스는 읽기 전용입니다. Claude가 INSERT, UPDATE, DELETE 등의 명령을 실행하려 하면 자동으로 거부됩니다. 데이터가 변경될 염려가 없습니다.

### 시나리오 3: 새 기능 만들기 (15분)

**목표**: Claude에게 새로운 API 기능을 직접 만들도록 요청합니다.

> 코드를 한 줄도 작성하지 않아도 됩니다. 원하는 기능을 말로 설명하면 Claude가 전부 작성합니다.

#### 순서

**1단계: 새 API 엔드포인트 요청**

```
안전 점검 보고서의 월별 추이를 보여주는 API 엔드포인트를 만들어줘. app/routes/safety.py에 추가하면 돼.
```

**예상 결과**: Claude가 다음 작업을 수행합니다.

- 기존 `safety.py` 파일을 읽고 코드 스타일을 파악
- 월별 보고서 건수를 집계하는 새로운 API 엔드포인트를 작성
- 작성한 코드의 동작 원리를 설명
- 파일을 직접 수정하거나 수정할 코드를 제시

> Claude가 파일을 수정할 때 확인 메시지가 나타납니다. **'y'를 입력**하면 실제로 파일이 변경됩니다.

**2단계: 코드 설명 요청**

```
방금 만든 코드가 정확히 어떻게 동작하는지 비개발자도 이해할 수 있게 설명해줘
```

**예상 결과**: Claude가 기술 용어를 최소화하여 다음을 설명합니다.

- 이 API가 어떤 데이터를 제공하는지
- 누가 이 API를 사용할 수 있는지
- 어떤 형태로 결과가 나오는지

**3단계: 추가 기능 요청 (선택)**

자유롭게 추가 기능을 요청해 보세요.

```
심각도별 필터링도 추가해줘. 예를 들어 critical만 볼 수 있게.
```

```
API 응답에 전월 대비 증감률도 포함해줘
```

> **💡 팁:** Claude가 만든 코드가 마음에 들지 않으면 "이 부분을 이렇게 바꿔줘"라고 다시 요청하면 됩니다. 대화를 이어가면서 점점 원하는 방향으로 수정할 수 있습니다.

### 시나리오 4: 문서 작성 (10분)

> 개발 문서, 보고서, 제안서 등 다양한 형식의 문서를 한국어로 작성할 수 있습니다.

#### 순서

**1단계: API 문서 작성**

```
이 프로젝트의 API 문서를 한국어로 작성해줘. 각 API의 URL, 파라미터, 응답 형식, 사용 예시를 포함해줘.
```

**예상 결과**: Claude가 프로젝트의 모든 API를 분석하여 다음을 포함하는 문서를 작성합니다.

- API 개요
- 각 엔드포인트의 상세 설명 (URL, HTTP 메서드, 파라미터)
- 요청/응답 예시
- 에러 코드 설명

**2단계: 비개발자용 보고서 작성**

```
이 안전관리 시스템의 기능과 현황을 비개발자 임원이 이해할 수 있는 1페이지 보고서로 작성해줘
```

**예상 결과**: Claude가 기술 용어를 배제한 경영진 보고서 형태의 문서를 작성합니다.

**3단계: 마크다운 파일로 저장 (선택)**

```
방금 작성한 보고서를 ~/workspace/report.md 파일로 저장해줘
```

> **💡 팁:** Claude가 작성한 문서를 파일로 저장하면 나중에 복사하여 다른 곳에서 활용할 수 있습니다.

아래는 시작점으로 활용할 수 있는 질문 예시입니다. 자신의 업무에 맞게 변형하여 질문하세요.

```
우리 부서에서 매주 수작업으로 안전 점검 현황을 엑셀로 정리하고 있어. 이걸 자동화하려면 어떤 시스템이 필요할까?
```

```
엑셀 파일에서 특정 조건에 맞는 데이터를 추출하고 요약하는 Python 스크립트를 만들어줘
```

```
주간 보고서를 자동으로 생성하는 시스템을 설계해줘. 데이터는 DB에서 가져오고, 결과는 이메일로 보내는 구조야.
```

## 4. Claude Code 활용 팁

### 질문을 잘 하는 법

Claude에게 좋은 결과를 얻으려면 **구체적으로** 요청하는 것이 중요합니다.

| 비효율적인 질문 | 효과적인 질문 |
|---------------|-------------|
| "코드 분석해줘" | "app/routes/safety.py의 코드를 리뷰하고 개선할 점을 알려줘" |
| "DB 조회해줘" | "최근 한 달간 심각도가 critical인 보고서를 날짜순으로 보여줘" |
| "문서 만들어줘" | "이 API의 사용법을 비개발자가 이해할 수 있게 한국어로 문서화해줘" |
| "자동화해줘" | "매주 월요일에 지난주 안전 점검 통계를 이메일로 보내는 스크립트를 만들어줘" |

### 핵심 원칙

1. **구체적으로 요청하세요**: 원하는 결과물의 형식, 범위, 대상 독자를 명시합니다.
2. **맥락을 제공하세요**: "우리 부서에서는...", "비개발자를 위해..."처럼 배경을 알려줍니다.
3. **단계적으로 요청하세요**: 한 번에 모든 것을 요청하기보다 대화를 이어가며 구체화합니다.

### Claude가 실수했을 때

Claude가 잘못된 결과를 줄 수 있습니다. 이때는 간단히 지적하면 됩니다.

```
아니야, 그게 아니라 [원하는 내용]을 해줘
```

```
방금 결과가 틀린 것 같아. 다시 확인해줘.
```

```
이 부분은 좋은데, 저 부분은 이렇게 바꿔줘
```

### 유용한 명령어

| 명령어 | 설명 |
|--------|------|
| `claude` | Claude Code 대화 시작 |
| `/help` | Claude Code 내장 도움말 보기 |
| `/clear` | 대화 기록 초기화 (새로운 주제를 시작할 때) |
| `Ctrl + C` | 현재 진행 중인 작업 중단 |
| `exit` 또는 `/exit` | Claude Code 종료 |

## 5. 자주 묻는 질문 (Q&A)

### 보안 관련

**Q: 우리 회사 데이터가 외부로 유출되나요?**

> A: 아닙니다. 모든 AI 처리는 **우리 회사의 AWS 계정 안에 있는 Bedrock 서비스**를 통해 이루어집니다. 데이터가 AWS 밖으로 나가지 않으며, Anthropic(Claude 개발사)도 우리 데이터에 접근할 수 없습니다. VPC 엔드포인트를 통한 내부 통신만 사용합니다.

**Q: 데이터베이스를 실수로 변경하면 어떻게 되나요?**

> A: 걱정하지 않으셔도 됩니다. 여러분이 접속하는 데이터베이스는 **읽기 전용(ReadOnly) 복제본**입니다. 데이터를 조회만 할 수 있고, 삽입(INSERT), 수정(UPDATE), 삭제(DELETE) 명령은 자동으로 거부됩니다.

**Q: 다른 참석자의 환경에 접근할 수 있나요?**

> A: 불가능합니다. 각 참석자는 완전히 격리된 독립 환경(Pod)을 사용합니다. 네트워크 정책(NetworkPolicy)으로 Pod 간 통신이 차단되어 있어 다른 사용자의 환경을 볼 수 없습니다.

### 비용 관련

**Q: 비용이 얼마나 발생하나요?**

> A: Bedrock은 사용량 기반 과금입니다. 오늘 실습의 경우 1인당 1시간 기준 약 $0.5 (약 650원) 수준으로 추정됩니다. 질문량과 응답 길이에 따라 다소 차이가 있을 수 있습니다.

**Q: AI를 많이 사용하면 비용이 급격히 늘어나나요?**

> A: 비용은 AI에게 보내는 텍스트 양과 AI가 생성하는 텍스트 양에 비례합니다. 오늘 실습 정도의 사용량에서는 비용이 크지 않습니다. 전사 도입 시에는 사용량 상한과 모니터링을 통해 비용을 관리할 계획입니다.

### 활용 관련

**Q: 코드를 모르는데 사용할 수 있나요?**

> A: 네, 가능합니다. Claude Code는 한국어 자연어로 대화하면 됩니다. "이 코드가 뭘 하는 거야?", "데이터베이스에서 이런 정보를 찾아줘"처럼 일상적인 말로 요청하면 Claude가 기술적인 부분을 처리합니다.

**Q: Claude가 틀린 답변을 하면 어떡하나요?**

> A: AI는 때때로 부정확한 답변을 할 수 있습니다. 이는 정상적인 현상입니다. 틀린 부분을 지적하면 Claude가 수정합니다. 중요한 의사결정에는 반드시 전문가의 검증을 거쳐야 합니다.

**Q: 우리 부서에 도입하려면 어떻게 해야 하나요?**

> A: 오늘 실습은 Phase 1(임원 체험)입니다. Phase 2에서 팀장급 50명으로 확대하고, 이후 실무자 상시 활용 환경을 구축할 계획입니다. 도입 관련 문의는 실습 후 안내되는 연락처로 접수해 주시기 바랍니다.

### 기술 관련

**Q: 어떤 AI 모델을 사용하나요?**

> A: Anthropic의 **Claude Sonnet 4.6** 모델을 AWS Bedrock을 통해 사용합니다. Bedrock은 AWS의 관리형 AI 서비스로, 별도 서버를 구축하지 않아도 API 호출 방식으로 AI 모델을 사용할 수 있습니다.

**Q: 실습 환경은 언제까지 유지되나요?**

> A: 실습 환경(Pod)은 **로그아웃하거나 최대 4시간이 지나면 자동 삭제**됩니다. 실습 중 작성하거나 수정한 파일은 환경 삭제 시 함께 사라지므로, 보관이 필요한 결과물은 별도로 복사해 두시기 바랍니다.

## 6. 트러블슈팅

### 문제 1: 터미널 화면이 열리지 않을 때

**증상**: 브라우저에서 URL을 입력했으나 빈 화면이거나 에러 메시지가 표시됩니다.

**해결 방법**:

1. 브라우저 주소창의 URL이 정확한지 다시 확인합니다.
2. 브라우저를 새로고침합니다 (F5 또는 Ctrl+R / Cmd+R).
3. 다른 브라우저(Chrome, Edge)로 시도합니다.
4. 위 방법으로도 해결되지 않으면 운영진에게 문의합니다.

### 문제 2: Claude Code가 응답하지 않을 때

**증상**: Claude에게 질문을 입력했으나 오랫동안 응답이 없습니다.

**해결 방법**:

1. 30초 이상 기다려 봅니다. 복잡한 요청은 처리 시간이 걸릴 수 있습니다.
2. `Ctrl + C`를 눌러 현재 작업을 중단합니다.
3. 같은 질문을 다시 입력하거나 더 간단하게 바꿔서 요청합니다.
4. 그래도 응답이 없으면 `/exit`으로 Claude Code를 종료한 뒤 `claude`로 다시 시작합니다.

### 문제 3: DB 접속 오류

**증상**: Claude가 데이터베이스에 접속하지 못한다고 합니다.

**해결 방법**:

1. Claude에게 다음과 같이 구체적으로 안내합니다.
   ```
   psql $DATABASE_URL 명령어로 DB에 접속할 수 있어. 이걸로 다시 시도해줘.
   ```
2. 직접 터미널에서 DB 접속을 확인합니다.
   ```bash
   psql $DATABASE_URL -c "SELECT 1;"
   ```
   결과가 `1`이 나오면 정상입니다. 에러가 나오면 운영진에게 문의합니다.

### 문제 4: "Permission denied" 또는 권한 오류

**증상**: Claude가 어떤 작업을 수행하려다 권한 오류가 발생합니다.

**해결 방법**:

1. 데이터베이스에 쓰기(INSERT, UPDATE, DELETE) 작업을 시도한 경우 정상적인 차단입니다. 읽기(SELECT) 작업만 가능합니다.
2. 파일 수정 권한 오류는 대부분 시스템 파일에 대한 접근 시도입니다. 샘플 프로젝트 폴더(`~/workspace/sample-project/`) 안에서 작업하도록 요청합니다.

### 문제 5: 세션 시간 초과

**증상**: 터미널이 갑자기 끊기거나 "세션이 만료되었습니다" 메시지가 표시됩니다.

**해결 방법**:

1. 세션은 최대 4시간 후 자동 종료됩니다.
2. 브라우저에서 다시 접속 URL을 입력하면 새로운 환경이 생성됩니다.
3. 이전 환경에서 작업하던 내용은 복구되지 않으니, 중요한 결과물은 미리 복사해 두세요.

### 문제 6: 화면이 깨지거나 글자가 겹칠 때

**증상**: 터미널 화면이 깨지거나 텍스트가 이상하게 표시됩니다.

**해결 방법**:

1. 브라우저 창의 크기를 조절(확대/축소)합니다.
2. `Ctrl + L`을 눌러 터미널 화면을 새로고침합니다.
3. 브라우저를 새로고침합니다 (F5).
